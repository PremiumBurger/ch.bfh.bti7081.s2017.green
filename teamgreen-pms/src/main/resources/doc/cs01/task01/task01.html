Task 01 - First analysis

In your team, identify

  1. target users
  2. key features (from the users' perspective)
  3. critical success factors

Target Users

Benutzer, welche dieses System bewirtschaften werden
# Rolle Jobbezeichnung Zugriff Rollenbeschreibung
1 Medical User Docotr, Nurse, Health visitor Vollzugriff Administratoren des Systems
2 Clinical Staff Receptionist Data-Input Verwaltet Termine mit Patienten
3 Medical Records Staff ? Data-Output Können Reports generieren für das Management

Key features

Hauptpunkte, welche dafür sprechen dieses System zu bauen
Rolle Features
Medical User
  • Medikamentierung / Rezepte - CRUD
  • Behandlungen - CRUD
  • System warnt falls Medikamente falsch dosiert verschrieben werden
Clinical Staff
  • Patientenverwaltung (Stammdatenpflege)
  • Patienten können als "obdachlos" gekennzeichnet werden
  • Patienten können als "gefährdet" gekennzeichnet werden
  • Auflisten von Ärzten / Kliniken welche für den Patienten X geeignet sind (auch unter berücksichtigung der Kennzeichnung "gefährdet")
  • Planen von Terminen - Patient X am dd.MM.yyyyT14:30:00 mit Arzt Y in Klinik Z
  • System verhindert Konsultationsüberschneidungen des selben Arztes
  • Reporte generieren (Anforderungen von Management)

Critical success factors

Müssen erfüllt sein, damit der Projektabschluss als erfolgreich angesehen werden kann
Faktor Beschreibung
Datenschutzregelung
  • Rechtliche Bestimmungen (Datenschutzgesetz, Gesundheitsgesetz)
  • Zugriffsberechtigungen
Zentralisierte Daten
  • Verfügbarkeit (Online Zugriff 7/24)
  • Sicherheit
