Task 07 - Domain Model and Sequence Diagram

For your application, design:

  1. a UML domain model covering the main classes

    • incl. association, inheritance, and aggregation relations
    • observe the general modeling guidelines and make use of Responsibility Driven Design (RDD) and use CRC-Cards as a tool.

  2. based on your domain model, create two UML sequence diagrams, one each for the following situations

    • when the user updates some specific information
    • when the system alerts the user about some specific event

  3. if necessary, refine your UML class model to include the attributes and operations needed for your sequence diagrams.

CRC-Cards

Class Name: Person

Purpose: Kapselt die Eigenschaften und Methoden einer Person

Responsibilities:
  • Hat eine eindeutige Nummer
  • Kennt seinen Vor- und Nachnamen
  • Kennt sein Geburtsdatum
  • Kennt seine Festnetz- und Mobilenummer
  • Kennt seine Mailadresse
  • Kennt seine AHV-Nummer
  • Kennt seine Haupt - und Zusatzadressen

Class Name: Patient

Purpose: Kapselt die Eigenschaften und Methoden eines Borderline-Patienten

Responsibilities:
  • Kennt seine behandelnden Ärzte
  • Kennt seine Healthvisitors
  • Kennt sein Journal
  • Kennt seine Termine
  • Kennt seine Folgelösung
  • Kann Folgelösung für sich selbst erstellen
Collaborators:
  • Ist eine Person
  • Kann von HealthVisitor aufgerufen und angezeigt werden

Class Name: Health Visitor

Purpose: Kapselt die Eigenschaften und Methoden eines Healthvisitor

Responsibilities:
  • Kennt seine Termine
  • Kennt seine Patienten
  • Kennt seine ausgelösten Alarme
Collaborators:
  • Ist eine Person

Class Name: Appointment

Purpose: Kapselt die Eigenschaften und Methoden eines Termins

Responsibilities:
  • Kennt seinen Patienten
  • Kennt seinen HealthVisitor
  • Kennt seine Verordnungen
  • Kann prüfen, ob Termin erinnert werden muss
Collaborators:
  • Wird von HealthVisitors verwaltet

Class Name: Follow up Solution

Purpose: Kapselt die Eigenschaften und Methoden einer Anschlusslösung

Responsibilities:
  • Kennt die ID der Externen Notschlafstelle
  • Kennt seinen Patienten
  • Kennt seinen Health Visitor
Collaborators:
  • Wird von Health Visitors verwaltet
