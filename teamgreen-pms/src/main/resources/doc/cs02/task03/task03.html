Task 03 - Elicitate Requirements

Given the description of the Personal Insulin Pump (PIP) in the CS2 introduction and your results from Tasks 1 and 2, try to elicitate the requirements.

  1. Start with high-level user requirements.
  2. Then elaborate high-level system requirements.
  3. Detail these system requirements into:
  • Functional Requirements
  • Non-Functional Requirements
  • Domain Requirements

User Requirements

  1. Das System muss den Blutzuckerspiegel ausgleichen können.
  2. Der Benutzer muss den Batteriestand verständlich auf dem Display ablesen können.
  3. Der Benutzer muss den Insulinvorrat ablesen verständlich auf dem Display ablesen können.
  4. Der Benutzer muss über Fehlfunktionen verständlich auf dem Display informiert werden.
  5. Der Benutzer kann eine History des Insulinspiegels und der Injektionen auf dem Display ablesen.

System Requirements

User Story:

1. Das System muss den Blutzuckerspiegel ausgleichen können.

Functional requirements:
  • Das System entnimmt dem Patienten in regelmässigen Abständen Blut.
  • Der Blutzuckerspiegel gemessen.
  • Der Prozessor errechnet aus dem Verlauf und den Messdaten den Insulinbedarf und injeziert bei bedarf.
Non Functional requirements:
  • Zeitlich und quantitativ opimale Injektion.
  • 24/7 Verfübar.
  • Batterielaufzeit von mindestens 14 Tage
Domain:
  • Muss der Internationalen Insulin Pumpen Norm nach FDA (:-) ) entsprechen
User Story:

2. Der Benutzer muss den Batteriestand verständlich auf dem Display ablesen können.

Functional requirements:
  • Die Batteriespannung wird ausgelesen und auf dem Display angezeigt.
  • Falls der Batteriestand in den kritischen bereich kommt, wird ein akustischer Alarm und eine Meldung auf dem Display ausgegeben
Non Functional requirements:
  • Die akustische Warnung soll bei 20% restspannung der Batterie erfolgen.
  • Eine grafische Warnung soll bereits bei 30% Restspannung auf dem Display ersichtlich sein
User Story:

3. Der Benutzer muss den Insulinvorrat ablesen verständlich auf dem Display ablesen können.

Functional requirements:
  • Jeweils nach der Injektion wird die verbleibende Restmenge an Insulin berechnet. Bei i unterschreiten des Grenzwertes wird ein Alarm mit einer Meldung auf dem Display ausgegeben.
Non Functional requirements:
  • Die erste grafische Warnung soll bei einer rest Dosis das einem Tages Dosis entspricht am Display ersichtlich sein
  • Eine akustische Warnung soll erfolgen wenn das rest Insulin nur noch für 1 injektion genügt
User Story:

4. Der Benutzer muss über Fehlfunktionen verständlich auf dem Display informiert werden.

Functional requirements:
  • Die Komponenten (Pumpe, Nadel, Controller..) sind mit Sensoren ausgestattet, die eine Fehlermeldung ausgeben, sobald ein Problem auftritt (z.B. Pumpen-Fehlfunktion, Verstopfungen in den Leitungen, Nadel ist nicht sauber appliziert).. Es erfolgt ein Alarm / eine Meldung auf dem Display.
Non Functional requirements:
  • Hinweise zur Behebung des Problems auf dem Display
User Story:

5. Der Benutzer kann eine History des Insulinspiegels und der Injektionen auf dem Display ablesen.

Functional requirements:
  • Folgende Daten aus dem letzten Injektionen sollen auf dem Display ersichtlich sein.
    1. Zeitpunkt der Injektion
    2. Insulin Dosis
    3. Blutzuckerspiegel
Non Functional requirements:
  • Saubere, übersichtliche graphische Darstellung auf dem Display
  • Die letzten 50 Injektions daten sollen im Log vorhanden sein
